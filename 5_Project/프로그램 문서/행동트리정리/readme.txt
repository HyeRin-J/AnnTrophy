참고 기획

<몬스터 타입>
1. 근거리 (근접형)
2. 원거리 (고정형)
3. 방패형 (근접형)
4. 도약형 (근접형)

<행동패턴>
1. 정찰상태 // Observe
-몬스터는 바라보는 방향으로 이동한다.
-벽에 닿을 경우 반대로 플립 후 이동한다.
-해당 상태를 반복한다.
(플랫폼에서 떨어질 수 있음)

2. 추격상태 // tracking
전이조건: 플레이어 인식
-근접: 정찰을 멈추고 플레이어를 따라간다.
(플레이어 피봇 좌표를 향해)
-원거리: 플레이어 인식 좌표로 탄을 날림
-인식범위에서 벗어난 후 상태변환 딜레이 변수값 이상으로 인식되지 않는 경우 다시 정찰상태로 돌아감.

3. 공격 // attack
전이조건: 추격 상태에서 공격 가능한 사거리 내 플레이어가 위치함
-공격 ani 실행 후 타격 시 PC의 -hp
-한번 공격 후 공격 딜레이동안 플레이어를 추격하는 상태로 넘어간다.

4. 사망 // Dead
전이조건: hp = 0
